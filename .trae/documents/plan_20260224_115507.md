# Piano di Implementazione CMS "Lavagna" con Astro e Keystatic

Questo piano descrive i passaggi per creare il tuo blog/CMS con un'interfaccia a lavagna e post-it, utilizzando Astro e Keystatic.

## 1. Setup del Progetto
*   Creare un nuovo progetto Astro nella directory corrente.
*   Installare le integrazioni necessarie: React (richiesto da Keystatic) e Tailwind CSS (per lo styling).
*   Pulire i file di esempio predefiniti.

## 2. Configurazione Keystatic (CMS)
*   Installare `@keystatic/astro` e `@keystatic/core`.
*   Configurare `keystatic.config.ts` con la modalit√† `storage: 'local'` (per salvare i file Markdown direttamente nella repo).
*   Definire la "Collection" per i **Post**:
    *   `title`: Titolo del post.
    *   `slug`: URL del post.
    *   `content`: Contenuto in Markdown/MDX.
    *   `date`: Data di pubblicazione.
    *   `color`: Colore del post-it (opzionale, es. giallo, rosa, azzurro).
*   Integrare Keystatic in `astro.config.mjs`.

## 3. Configurazione Content Collections di Astro
*   Configurare `src/content/config.ts` per far corrispondere lo schema di Astro a quello di Keystatic.
*   Questo permette ad Astro di leggere in modo efficiente i file generati da Keystatic.

## 4. Design e Layout "Lavagna"
*   Configurare Tailwind CSS.
*   Aggiungere un font "Handwritten" (es. da Google Fonts come "Kalam" o "Indie Flower") per simulare la scrittura a mano.
*   Creare un background CSS che simuli la texture di una lavagna (colore scuro verdone/grigio scuro).
*   Creare il componente `PostIt`:
    *   Aspetto quadrato/rettangolare.
    *   Ombra per effetto sollevato.
    *   Font handwritten.
    *   Effetto "pin" (puntina) o nastro adesivo in alto (opzionale).

## 5. Sviluppo Pagine
*   **Homepage (`index.astro`)**:
    *   Recuperare tutti i post dalla collection.
    *   Visualizzarli come una griglia sparsa di Post-it sulla lavagna.
*   **Pagina Articolo (`posts/[slug].astro`)**:
    *   Layout di dettaglio. Potrebbe sembrare un foglio di carta appoggiato sulla lavagna o un post-it ingrandito.
    *   Rendering del contenuto Markdown.

## 6. Verifica
*   Avviare il server di sviluppo.
*   Accedere alla dashboard di Admin di Keystatic (`/keystatic`) per creare un post di prova.
*   Verificare che il post appaia sulla "lavagna" in homepage.
